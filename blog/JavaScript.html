Ruby classes vs. JavaScript constructor functions


Recently, I developed a command-line text-based basketball simulator game. I actually ported the code from an older software program I wrote as a teenager to JavaScript. The older program was written in a non-Object Oriented Language and thus I needed to create classes in JavaScript to get all the benefits.

Because I wanted many teams to be created in the game, I used a JavaScript constructor to easily initialize a team and it's Print Roster function.
After I finished the first version of this game, I decided to port it over to Ruby, which utilizes classes as everything is an object. I needed to remember the proper syntax for class variables and added an @ to each of them. In JavaScript, the same thing is accomplished by using the this keyword.

In Ruby, you create an initialize method inside the class but in JavaScript all attributes and functions are contained within the constructor function. I believe that Ruby allows for easier expansion of classes in this way.

You can see my code below for a great comparison of the two.
